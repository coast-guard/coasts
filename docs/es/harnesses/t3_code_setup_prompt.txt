Estás configurando la habilidad Coasts para T3 Code en este proyecto. Ejecuta todos
los comandos desde la raíz del proyecto.

## Paso 1: Verificar Coast CLI

Ejecuta este comando:

  coast --version

Si no se encuentra el comando `coast`, detente aquí y dile al usuario:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

No continúes hasta que el CLI esté disponible.

## Paso 2: Obtener el contenido de la habilidad

Ejecuta este comando:

  coast skills-prompt

La salida tiene dos partes:

- **Reglas de Coast Runtime** — todo desde el inicio hasta (pero sin incluir) la línea que comienza con `---`
- **Habilidad Coasts** — todo desde el bloque de frontmatter `---` en adelante (incluyendo las líneas `---` y todo lo que sigue)

Guarda ambas partes para los pasos siguientes.

## Paso 3: Preguntar al usuario

T3 Code admite múltiples proveedores. Pregunta al usuario: ¿qué proveedores usas
en T3 Code?

- **Codex** (OpenAI)
- **Claude** (Anthropic)
- **Ambos**
- **Otro** — si el usuario nombra un proveedor diferente, ejecuta
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` y
  `coast docs --path harnesses/README.md` para determinar dónde ese proveedor
  espera las instrucciones del proyecto y las habilidades, y luego sigue el mismo patrón

## Paso 4: Colocar archivos

La configuración de T3 Code siempre es a nivel de proyecto. Si un archivo de destino ya existe, añade
la sección Coast Runtime en lugar de sobrescribirla, pero primero verifica si ya
hay una sección `# Coast Runtime` y omítela si es así.

**Si el usuario seleccionó Codex (o ambos):**
- Añade las reglas de Coast Runtime a `AGENTS.md` en la raíz del proyecto
- Escribe la habilidad Coasts en `.agents/skills/coasts/SKILL.md` (a nivel de proyecto)
- También escribe la habilidad Coasts en `~/.codex/skills/coasts/SKILL.md` (respaldo
  global — es posible que T3 Code todavía no examine `.agents/skills/` a nivel de proyecto)
- Crea `.agents/skills/coasts/agents/openai.yaml` con este contenido:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

**Si el usuario seleccionó Claude (o ambos):**
- Añade las reglas de Coast Runtime a `CLAUDE.md` en la raíz del proyecto
- Escribe la habilidad Coasts en `.claude/skills/coasts/SKILL.md` (a nivel de proyecto)
- También escribe la habilidad Coasts en `~/.claude/skills/coasts/SKILL.md` (respaldo
  global — es posible que T3 Code todavía no examine `.claude/skills/` a nivel de proyecto de forma confiable)

**Nota:** Es posible que T3 Code todavía no cargue habilidades a nivel de proyecto desde `.agents/skills/` o
`.claude/skills/`. Las reglas de `AGENTS.md` y `CLAUDE.md` siguen aplicándose en
cada tarea de todos modos. El respaldo global `~/.codex/skills/coasts/` garantiza
que el proveedor Codex pueda encontrar la habilidad.

## Paso 5: Actualizar el Coastfile

Lee el `Coastfile` en la raíz del proyecto. Revisa el campo `worktree_dir` en la
sección `[coast]`. También lee el campo `name`; lo necesitarás para construir la
ruta del worktree.

El directorio de worktree de T3 Code es `~/.t3/worktrees/<name>`, donde `<name>` es
el valor del campo `name` en el Coastfile.

Si esa ruta **aún no** aparece en `worktree_dir`:

- Si `worktree_dir` es una sola cadena, conviértela en un arreglo y añade la
  ruta de T3 Code. Por ejemplo, `worktree_dir = ".worktrees"` se convierte en
  `worktree_dir = [".worktrees", "~/.t3/worktrees/my-app"]`.
- Si `worktree_dir` ya es un arreglo, añade la ruta de T3 Code.
- Si `worktree_dir` no está presente en absoluto, agrega
  `worktree_dir = [".worktrees", "~/.t3/worktrees/<name>"]` usando el
  nombre real del proyecto.

Esta es una ruta externa, así que si ya hay una instancia de Coast ejecutándose para este
proyecto, debe recrearse con `coast run` para que el nuevo bind mount surta
efecto. Díselo al usuario.

## Paso 6: Confirmar

Después de colocar los archivos, muestra al usuario un resumen de lo que se creó y dónde.
Explica que la copia global `~/.codex/skills/coasts/` es una solución temporal para
que T3 Code aún no cargue habilidades a nivel de proyecto.

Dile al usuario: **Reinicia T3 Code** para que los cambios en la habilidad y las reglas surtan
efecto.
