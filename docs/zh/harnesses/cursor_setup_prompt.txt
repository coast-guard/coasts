你正在为此项目中的 Cursor 设置 Coasts 技能。请从项目根目录运行所有
命令。

## 第 1 步:检查 Coast CLI

运行以下命令:

  coast --version

如果未找到 `coast` 命令，请在此停止并告诉用户:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

在 CLI 可用之前不要继续。

## 第 2 步:获取技能内容

运行以下命令:

  coast skills-prompt

输出包含两部分:

- **Coast Runtime 规则** — 从开头到以 `---` 开始的那一行之前（但不包括该行）的所有内容
- **Coasts 技能** — 从 `---` frontmatter 块开始往后的所有内容（包括 `---` 行以及其后的所有内容）

保存这两部分，以供下面的步骤使用。

## 第 3 步:询问用户

询问用户:我应该将其设置为**全局**（此机器上的每个项目中都可用）还是**仅用于此项目**？

## 第 4 步:放置文件

如果目标文件已存在，请追加 Coast Runtime 部分而不是覆盖——但要先检查是否已经存在 `# Coast Runtime` 部分，如果存在则跳过。

**全局设置:**
- 将 Coast Runtime 规则追加到用户的全局 Cursor 规则中
- 将 Coasts 技能写入 `~/.cursor/skills/coasts/SKILL.md`

**项目设置:**
- 将 Coast Runtime 规则追加到项目根目录的 `AGENTS.md` 中（如果用户更喜欢 Cursor 原生规则，则写入 `.cursor/rules/coast.md`——询问他们）
- 将 Coasts 技能写入 `.cursor/skills/coasts/SKILL.md`

## 第 5 步:更新 Coastfile

读取项目根目录中的 `Coastfile`。查看
`[coast]` 部分中的 `worktree_dir` 字段。还要读取 `name` 字段——你将需要它来构造
worktree 路径。

Cursor 的 worktree 目录是 `~/.cursor/worktrees/<name>`，其中 `<name>` 是
Coastfile 中 `name` 字段的值。

如果该路径**尚未**列在 `worktree_dir` 中:

- 如果 `worktree_dir` 是单个字符串，将其转换为数组并追加
  Cursor 路径。例如，`worktree_dir = ".worktrees"` 变为
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/my-app"]`。
- 如果 `worktree_dir` 已经是数组，则将 Cursor 路径追加到其中。
- 如果 `worktree_dir` 完全不存在，则添加
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/<name>"]`，并使用
  实际的项目名称。

这是一个外部路径，因此如果此项目的 Coast 实例已经在运行，
则必须使用 `coast run` 重新创建它，新的 bind mount 才会生效。
请告知用户这一点。

## 第 6 步:确认

放置文件后，向用户显示已创建内容及其位置的摘要。

告诉用户:**重启 Cursor**，使技能和规则更改生效。
