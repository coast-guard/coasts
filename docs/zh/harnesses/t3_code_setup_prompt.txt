你正在为此项目中的 T3 Code 设置 Coasts 技能。请从项目根目录运行所有
命令。

## Step 1: 检查 Coast CLI

运行此命令:

  coast --version

如果找不到 `coast` 命令，请在此停止并告诉用户:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

在 CLI 可用之前不要继续。

## Step 2: 获取技能内容

运行此命令:

  coast skills-prompt

输出有两部分:

- **Coast Runtime 规则** — 从开头直到（但不包括）以 `---` 开头的那一行之前的所有内容
- **Coasts 技能** — 从 `---` frontmatter 代码块开始往后的所有内容（包括 `---` 行以及其后的所有内容）

请为下面的步骤保存这两部分内容。

## Step 3: 询问用户

T3 Code 支持多个提供商。询问用户:你在 T3 Code 中使用哪些提供商？

- **Codex** (OpenAI)
- **Claude** (Anthropic)
- **Both**
- **Other** — 如果用户提到不同的提供商，运行
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` 和
  `coast docs --path harnesses/README.md` 以确定该提供商期望将项目说明和技能放在哪里，然后遵循相同的模式

## Step 4: 放置文件

T3 Code 的设置始终是项目级别的。如果目标文件已经存在，请追加
Coast Runtime 部分而不是覆盖——但首先检查是否已经存在
`# Coast Runtime` 部分，如果存在则跳过。

**如果用户选择了 Codex（或 both）:**
- 将 Coast Runtime 规则追加到项目根目录的 `AGENTS.md`
- 将 Coasts 技能写入 `.agents/skills/coasts/SKILL.md`（项目级别）
- 还要将 Coasts 技能写入 `~/.codex/skills/coasts/SKILL.md`（全局
  回退方案——T3 Code 可能尚未扫描项目级别的 `.agents/skills/`）
- 创建 `.agents/skills/coasts/agents/openai.yaml`，内容如下:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

**如果用户选择了 Claude（或 both）:**
- 将 Coast Runtime 规则追加到项目根目录的 `CLAUDE.md`
- 将 Coasts 技能写入 `.claude/skills/coasts/SKILL.md`（项目级别）
- 还要将 Coasts 技能写入 `~/.claude/skills/coasts/SKILL.md`（全局
  回退方案——T3 Code 可能尚未可靠地扫描项目级别的 `.claude/skills/`）

**注意:** T3 Code 可能尚未从 `.agents/skills/` 或
`.claude/skills/` 加载项目级别的技能。不过，`AGENTS.md` 和 `CLAUDE.md` 中的规则仍然会在
每个任务中生效。全局 `~/.codex/skills/coasts/` 回退方案可确保
Codex 提供商能够找到该技能。

## Step 5: 更新 Coastfile

读取项目根目录中的 `Coastfile`。查看
`[coast]` 部分中的 `worktree_dir` 字段。还要读取 `name` 字段——你将需要它来构造
worktree 路径。

T3 Code 的 worktree 目录是 `~/.t3/worktrees/<name>`，其中 `<name>` 是
Coastfile 中 `name` 字段的值。

如果该路径**尚未**列在 `worktree_dir` 中:

- 如果 `worktree_dir` 是单个字符串，将其转换为数组并追加
  T3 Code 路径。例如，`worktree_dir = ".worktrees"` 变为
  `worktree_dir = [".worktrees", "~/.t3/worktrees/my-app"]`。
- 如果 `worktree_dir` 已经是数组，则将 T3 Code 路径追加到其中。
- 如果根本不存在 `worktree_dir`，则添加
  `worktree_dir = [".worktrees", "~/.t3/worktrees/<name>"]`，其中使用
  实际的项目名称。

这是一个外部路径，因此如果该项目的 Coast 实例已经在运行，
则必须使用 `coast run` 重新创建它，新的绑定挂载才能生效。
请将此事告知用户。

## Step 6: 确认

放置文件后，向用户展示创建了什么以及创建到了哪里。
解释全局 `~/.codex/skills/coasts/` 副本是一个变通方案，因为
T3 Code 尚未加载项目级别的技能。

告诉用户:**重启 T3 Code**，以使技能和规则的更改生效。
