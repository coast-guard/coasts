이 프로젝트에서 T3 Code용 Coasts 스킬을 설정하고 있습니다. 모든
명령은 프로젝트 루트에서 실행하세요.

## Step 1: Coast CLI 확인

다음 명령을 실행하세요:

  coast --version

`coast` 명령을 찾을 수 없다면, 여기서 중단하고 사용자에게 다음과 같이 알려주세요:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

CLI를 사용할 수 있을 때까지 계속 진행하지 마세요.

## Step 2: 스킬 콘텐츠 가져오기

다음 명령을 실행하세요:

  coast skills-prompt

출력은 두 부분으로 나뉩니다:

- **Coast Runtime rules** — 시작부터 `---`로 시작하는 줄 직전까지의 모든 내용
- **Coasts skill** — `---` 프론트매터 블록부터 이후의 모든 내용 (`---` 줄과 그 이후의 모든 내용 포함)

아래 단계에서 사용할 수 있도록 두 부분을 모두 저장하세요.

## Step 3: 사용자에게 묻기

T3 Code는 여러 제공자를 지원합니다. 사용자에게 물어보세요: T3 Code에서
어떤 제공자를 사용하시나요?

- **Codex** (OpenAI)
- **Claude** (Anthropic)
- **Both**
- **Other** — 사용자가 다른 제공자를 언급하면,
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` 및
  `coast docs --path harnesses/README.md`를 실행하여 해당 제공자가
  프로젝트 지침과 스킬을 어디에서 기대하는지 확인한 다음, 동일한 패턴을 따르세요

## Step 4: 파일 배치

T3 Code 설정은 항상 프로젝트 수준입니다. 대상 파일이 이미 존재하면,
덮어쓰는 대신 Coast Runtime 섹션을 덧붙이세요 — 하지만 먼저
`# Coast Runtime` 섹션이 이미 있는지 확인하고, 있다면 건너뛰세요.

**사용자가 Codex를 선택한 경우(또는 both):**
- 프로젝트 루트의 `AGENTS.md`에 Coast Runtime 규칙을 덧붙이세요
- Coasts 스킬을 `.agents/skills/coasts/SKILL.md`에 작성하세요 (프로젝트 수준)
- Coasts 스킬을 `~/.codex/skills/coasts/SKILL.md`에도 작성하세요 (전역
  폴백 — T3 Code가 아직 프로젝트 수준 `.agents/skills/`를 스캔하지 못할 수 있음)
- 다음 내용으로 `.agents/skills/coasts/agents/openai.yaml`을 생성하세요:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

**사용자가 Claude를 선택한 경우(또는 both):**
- 프로젝트 루트의 `CLAUDE.md`에 Coast Runtime 규칙을 덧붙이세요
- Coasts 스킬을 `.claude/skills/coasts/SKILL.md`에 작성하세요 (프로젝트 수준)
- Coasts 스킬을 `~/.claude/skills/coasts/SKILL.md`에도 작성하세요 (전역
  폴백 — T3 Code가 아직 프로젝트 수준 `.claude/skills/`를 안정적으로 스캔하지 못할 수 있음)

**참고:** T3 Code는 아직 `.agents/skills/` 또는
`.claude/skills/`의 프로젝트 수준 스킬을 로드하지 못할 수 있습니다. `AGENTS.md`와 `CLAUDE.md`의 규칙은 그래도
모든 작업에 항상 적용됩니다. 전역 `~/.codex/skills/coasts/` 폴백은
Codex 제공자가 스킬을 찾을 수 있도록 보장합니다.

## Step 5: Coastfile 업데이트

프로젝트 루트의 `Coastfile`을 읽으세요. `[coast]` 섹션의 `worktree_dir` 필드를
확인하세요. 또한 `name` 필드도 읽으세요 — 워크트리 경로를 구성하는 데 필요합니다.

T3 Code 워크트리 디렉터리는 `~/.t3/worktrees/<name>`이며, 여기서 `<name>`은
Coastfile의 `name` 필드 값입니다.

해당 경로가 `worktree_dir`에 **아직** 나열되어 있지 않다면:

- `worktree_dir`가 단일 문자열이면, 배열로 변환하고
  T3 Code 경로를 덧붙이세요. 예를 들어, `worktree_dir = ".worktrees"`는
  `worktree_dir = [".worktrees", "~/.t3/worktrees/my-app"]`가 됩니다.
- `worktree_dir`가 이미 배열이면, 여기에 T3 Code 경로를 덧붙이세요.
- `worktree_dir`가 아예 없으면,
  `worktree_dir = [".worktrees", "~/.t3/worktrees/<name>"]`를
  실제 프로젝트 이름을 사용하여 추가하세요.

이것은 외부 경로이므로, 이 프로젝트에 대해 Coast 인스턴스가 이미 실행 중이라면
새 bind mount가 적용되도록 `coast run`으로 다시 생성해야 합니다.
이 점을 사용자에게 알려주세요.

## Step 6: 확인

파일 배치가 끝나면, 무엇이 생성되었고 어디에 생성되었는지 사용자에게
요약해서 보여주세요. 전역 `~/.codex/skills/coasts/` 사본은
T3 Code가 아직 프로젝트 수준 스킬을 로드하지 못하는 문제를 위한
우회책이라고 설명하세요.

사용자에게 다음을 알려주세요: 스킬 및 규칙 변경 사항이 적용되도록
**T3 Code를 다시 시작하세요**.
