이 프로젝트에서 Cursor용 Coasts 스킬을 설정하고 있습니다. 모든
명령은 프로젝트 루트에서 실행하세요.

## Step 1: Coast CLI 확인

이 명령을 실행하세요:

  coast --version

`coast` 명령을 찾을 수 없으면, 여기서 중단하고 사용자에게 다음과 같이 알리세요:

  "Coast CLI가 설치되어 있지 않습니다. 먼저 설치하세요: https://coasts.dev/docs/getting-started"

CLI를 사용할 수 있기 전까지는 계속 진행하지 마세요.

## Step 2: 스킬 콘텐츠 가져오기

이 명령을 실행하세요:

  coast skills-prompt

출력은 두 부분으로 구성됩니다:

- **Coast Runtime 규칙** — 시작부터 `---`로 시작하는 줄 직전까지의 모든 내용
- **Coasts 스킬** — `---` 프런트매터 블록부터 이후의 모든 내용 (`---` 줄과 그 뒤의 모든 내용 포함)

아래 단계에 사용할 수 있도록 두 부분을 모두 저장하세요.

## Step 3: 사용자에게 묻기

사용자에게 물어보세요: 이것을 **전역으로** 설정할까요(이 컴퓨터의 모든 프로젝트에서 사용 가능) 아니면 **이 프로젝트에만** 설정할까요?

## Step 4: 파일 배치

대상 파일이 이미 존재하는 경우, 덮어쓰지 말고 Coast Runtime 섹션을 덧붙이세요. 단, 먼저 `# Coast Runtime` 섹션이 이미 있는지 확인하고, 있으면 건너뛰세요.

**전역 설정:**
- Coast Runtime 규칙을 사용자의 전역 Cursor 규칙에 덧붙이세요
- Coasts 스킬을 `~/.cursor/skills/coasts/SKILL.md`에 작성하세요

**프로젝트 설정:**
- Coast Runtime 규칙을 프로젝트 루트의 `AGENTS.md`에 덧붙이세요(또는 사용자가 Cursor 네이티브 규칙을 선호한다면 `.cursor/rules/coast.md` — 사용자에게 물어보세요)
- Coasts 스킬을 `.cursor/skills/coasts/SKILL.md`에 작성하세요

## Step 5: Coastfile 업데이트

프로젝트 루트의 `Coastfile`을 읽으세요. `[coast]` 섹션의 `worktree_dir` 필드를
확인하세요. 또한 `name` 필드도 읽으세요 — 워크트리 경로를 구성하는 데
필요합니다.

Cursor 워크트리 디렉터리는 `~/.cursor/worktrees/<name>`이며, 여기서 `<name>`은
Coastfile의 `name` 필드 값입니다.

해당 경로가 아직 `worktree_dir`에 나열되어 있지 않다면:

- `worktree_dir`가 단일 문자열인 경우, 배열로 변환하고
  Cursor 경로를 추가하세요. 예를 들어, `worktree_dir = ".worktrees"`는
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/my-app"]`가 됩니다.
- `worktree_dir`가 이미 배열인 경우, Cursor 경로를 여기에 추가하세요.
- `worktree_dir`가 아예 없는 경우,
  실제 프로젝트 이름을 사용하여
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/<name>"]`를 추가하세요.

이것은 외부 경로이므로, 이 프로젝트에 대해 이미 실행 중인 Coast 인스턴스가 있다면
새 바인드 마운트가 적용되도록 `coast run`으로 다시 생성해야 합니다.
이 점을 사용자에게 알려주세요.

## Step 6: 확인

파일을 배치한 후, 무엇이 생성되었고 어디에 생성되었는지 사용자에게 요약해서 보여주세요.

사용자에게 다음을 알리세요: 변경된 스킬과 규칙이 적용되려면 **Cursor를 다시 시작**하세요.
