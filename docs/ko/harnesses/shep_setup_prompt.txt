이 프로젝트에서 Shep용 Coasts 스킬을 설정하고 있습니다. 모든
명령은 프로젝트 루트에서 실행하세요.

## Step 1: Coast CLI 확인

다음 명령을 실행하세요:

  coast --version

`coast` 명령을 찾을 수 없으면, 여기서 중단하고 사용자에게 다음과 같이 알려주세요:

  "Coast CLI가 설치되어 있지 않습니다. 먼저 설치하세요: https://coasts.dev/docs/getting-started"

CLI를 사용할 수 있을 때까지 계속 진행하지 마세요.

## Step 2: 스킬 콘텐츠 가져오기

다음 명령을 실행하세요:

  coast skills-prompt

출력은 두 부분으로 나뉩니다:

- **Coast Runtime 규칙** — 시작부터 `---`로 시작하는 줄 **직전까지의** 모든 내용
- **Coasts 스킬** — `---` 프런트매터 블록부터 이후의 모든 내용 (`---` 줄과 그 뒤의 모든 내용 포함)

아래 단계들을 위해 두 부분을 모두 저장하세요.

## Step 3: 파일 배치

Shep는 Claude Code를 래핑하므로, Claude Code 파일 레이아웃을 사용하세요. 대상 파일이
이미 존재하는 경우, 덮어쓰지 말고 Coast Runtime 섹션을 추가하세요. 하지만
먼저 `# Coast Runtime` 섹션이 이미 있는지 확인하고, 있으면 건너뛰세요.

- 프로젝트 루트의 `CLAUDE.md`에 Coast Runtime 규칙을 추가합니다
- Coasts 스킬을 `.agents/skills/coasts/SKILL.md`에 작성합니다 (또는
  프로젝트가 이미 해당 레이아웃을 사용하는 경우 `.claude/skills/coasts/SKILL.md`)

## Step 4: Coastfile 업데이트

프로젝트 루트의 `Coastfile`을 읽으세요. `[coast]` 섹션의
`worktree_dir` 필드를 확인하세요.

`~/.shep/repos/*/wt`가 `worktree_dir`에 **아직** 포함되어 있지 않다면:

- `worktree_dir`가 단일 문자열이면, 배열로 변환하고
  `~/.shep/repos/*/wt`를 추가하세요. 예를 들어, `worktree_dir = ".worktrees"`는
  `worktree_dir = [".worktrees", "~/.shep/repos/*/wt"]`가 됩니다.
- `worktree_dir`가 이미 배열이라면, 여기에 `~/.shep/repos/*/wt`를 추가하세요.
- `worktree_dir`가 아예 없다면,
  `worktree_dir = [".worktrees", "~/.shep/repos/*/wt"]`를 추가하세요.

`*`는 저장소별 해시 디렉터리와 일치하는 glob 패턴입니다. Coasts는
런타임에 이를 확장합니다.

이것은 외부 경로이므로, 이 프로젝트에 대해 Coast 인스턴스가 이미 실행 중이라면
새 bind mount가 적용되도록 `coast run`으로 다시 생성해야 합니다. 이 점을 사용자에게 알려주세요.

## Step 5: 확인

파일을 배치한 후, 무엇이 생성되었고 어디에 생성되었는지 사용자에게 요약해서 보여주세요.

사용자에게 다음을 알려주세요: **에디터를 종료했다가 다시 여세요** 그래야 스킬과 `CLAUDE.md`
변경 사항이 적용됩니다.
