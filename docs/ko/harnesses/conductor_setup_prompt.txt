이 프로젝트에서 Conductor용 Coasts 스킬을 설정하고 있습니다. 모든
명령은 프로젝트 루트에서 실행하세요.

## Step 1: Coast CLI 확인

다음 명령을 실행하세요:

  coast --version

`coast` 명령을 찾을 수 없으면 여기서 중단하고 사용자에게 다음과 같이 알리세요:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

CLI를 사용할 수 있을 때까지 계속 진행하지 마세요.

## Step 2: 스킬 내용 가져오기

다음 명령을 실행하세요:

  coast skills-prompt

출력은 두 부분으로 구성됩니다:

- **Coast Runtime 규칙** — 시작부터 `---`로 시작하는 줄 직전까지의 모든 내용
- **Coasts 스킬** — `---` 프론트매터 블록부터 이후의 모든 내용 (`---` 줄과 그 이후의 모든 내용 포함)

아래 단계들을 위해 두 부분을 모두 저장하세요.

## Step 3: 사용자에게 묻기

사용자에게 물어보세요: Conductor에서 어떤 제공자를 사용하시나요?

- **Claude** (Anthropic)
- **Codex** (OpenAI)
- **Both**
- **Other** — 사용자가 다른 제공자를 말하면,
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` 및
  `coast docs --path harnesses/README.md`를 실행하여 해당 제공자가
  프로젝트 지침과 스킬을 어디에 두기를 기대하는지 확인한 다음, 같은 패턴을 따르세요

## Step 4: 파일 배치

Conductor 설정은 항상 프로젝트 수준입니다. 대상 파일이 이미 존재하면 덮어쓰지 말고
Coast Runtime 섹션을 추가하세요. 단, 먼저
`# Coast Runtime` 섹션이 이미 있는지 확인하고 있으면 건너뛰세요.

**사용자가 Claude(또는 both)를 선택한 경우:**
- 프로젝트 루트의 `CLAUDE.md`에 Coast Runtime 규칙을 추가하세요
- Coasts 스킬을 `.claude/skills/coasts/SKILL.md`에 작성하세요

**사용자가 Codex(또는 both)를 선택한 경우:**
- 프로젝트 루트의 `AGENTS.md`에 Coast Runtime 규칙을 추가하세요
- Coasts 스킬을 `.agents/skills/coasts/SKILL.md`에 작성하세요
- 다음 내용으로 `.agents/skills/coasts/agents/openai.yaml`을 생성하세요:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

## Step 5: 확인

파일을 배치한 후, 무엇이 생성되었고 어디에 생성되었는지 사용자에게 요약해서 보여주세요.

변경 사항이 적용되도록 사용자에게 다음을 알려주세요: **Conductor를 완전히 종료한 뒤 다시 여세요**.
`/coasts` 명령이 바로 나타나지 않으면, 다시 종료했다가 다시 여세요.

## Step 6: Coastfile 업데이트

프로젝트 루트의 `Coastfile`을 읽으세요. `[coast]` 섹션의 `worktree_dir`
필드를 확인하세요. 또한 `name` 필드도 읽으세요 — 워크트리 경로를 구성하는 데 필요합니다.

Conductor 워크트리 디렉터리는 `~/conductor/workspaces/<name>`이며,
여기서 `<name>`은 Coastfile의 `name` 필드 값입니다.

해당 경로가 아직 `worktree_dir`에 나열되어 있지 않다면:

- `worktree_dir`가 단일 문자열이면 배열로 변환하고
  Conductor 경로를 추가하세요. 예를 들어, `worktree_dir = ".worktrees"`는
  `worktree_dir = [".worktrees", "~/conductor/workspaces/my-app"]`가 됩니다.
- `worktree_dir`가 이미 배열이면 Conductor 경로를 여기에 추가하세요.
- `worktree_dir`가 전혀 없으면 실제 프로젝트 이름을 사용해
  `worktree_dir = [".worktrees", "~/conductor/workspaces/<name>"]`를 추가하세요.

이것은 외부 경로이므로, 이 프로젝트에 대해 이미 실행 중인 Coast 인스턴스가 있다면
새 바인드 마운트가 적용되도록 `coast run`으로 다시 생성해야 합니다.
이 점을 사용자에게 알려주세요.

## Step 7: 새 파일 커밋

Conductor는 각 세션을 격리된 git worktree에서 실행합니다. 커밋되지 않은 파일은
새 세션으로 유지되지 않습니다. 방금 생성한 파일이 앞으로의 모든 워크스페이스에서
사용 가능하도록 커밋하세요:

  git add CLAUDE.md .claude/ AGENTS.md .agents/ 2>/dev/null; git status

어떤 파일이 스테이징되었는지 사용자에게 보여주고, 커밋하기 전에 확인을 요청하세요.
커밋 메시지는 `[dh] feat: add Coasts skill for Conductor`와 같은 형식을 사용하세요.
