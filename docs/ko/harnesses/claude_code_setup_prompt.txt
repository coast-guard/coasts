이 프로젝트에서 Claude Code용 Coasts 스킬을 설정하고 있습니다. 모든
명령은 프로젝트 루트에서 실행하세요.

## Step 1: Coast CLI 확인

다음 명령을 실행하세요:

  coast --version

`coast` 명령을 찾을 수 없으면, 여기서 중지하고 사용자에게 다음과 같이 알려주세요:

  "Coast CLI가 설치되어 있지 않습니다. 먼저 설치하세요: https://coasts.dev/docs/getting-started"

CLI를 사용할 수 있을 때까지 계속 진행하지 마세요.

## Step 2: 스킬 콘텐츠 가져오기

다음 명령을 실행하세요:

  coast skills-prompt

출력에는 두 부분이 있습니다:

- **Coast Runtime 규칙** — 시작부터 `---`로 시작하는 줄 직전까지의 모든 내용
- **Coasts 스킬** — `---` 프론트매터 블록부터 이후의 모든 내용 (`---` 줄과 그 뒤의 모든 내용 포함)

아래 단계들을 위해 두 부분을 모두 저장하세요.

## Step 3: 사용자에게 묻기

사용자에게 물어보세요: 이것을 **전역으로** 설정할까요(이 머신의 모든 프로젝트에서 사용 가능) 아니면 **이 프로젝트에만** 설정할까요?

## Step 4: 파일 배치

대상 파일이 이미 존재하는 경우, 덮어쓰는 대신 Coast Runtime 섹션을 추가하세요 — 하지만 먼저 `# Coast Runtime` 섹션이 이미 있는지 확인하고, 있으면 건너뛰세요.

**전역 설정:**
- Coast Runtime 규칙을 `~/.claude/CLAUDE.md`에 추가
- Coasts 스킬을 `~/.claude/skills/coasts/SKILL.md`에 작성

**프로젝트 설정:**
- Coast Runtime 규칙을 프로젝트 루트의 `CLAUDE.md`에 추가
- Coasts 스킬을 `.claude/skills/coasts/SKILL.md`에 작성

## Step 5: Coastfile 업데이트

프로젝트 루트의 `Coastfile`을 읽으세요. `worktree_dir` 필드를
`[coast]` 섹션에서 확인하세요.

`.claude/worktrees`가 `worktree_dir`에 **아직** 포함되어 있지 않다면:

- `worktree_dir`가 단일 문자열이면, 배열로 변환한 뒤
  `.claude/worktrees`를 추가하세요. 예를 들어, `worktree_dir = ".worktrees"`는
  `worktree_dir = [".worktrees", ".claude/worktrees"]`가 됩니다.
- `worktree_dir`가 이미 배열이면, 여기에 `.claude/worktrees`를 추가하세요.
- `worktree_dir`가 아예 없다면,
  `worktree_dir = [".worktrees", ".claude/worktrees"]`를 추가하세요.

이것은 상대 경로(프로젝트 루트 내부)이므로 컨테이너 재생성은
필요하지 않습니다 — 변경 사항은 worktree 목록에 즉시 반영됩니다.

## Step 6: 확인

파일을 배치한 후, 무엇이 생성되었고 어디에 생성되었는지에 대한 요약을 사용자에게 보여주세요.

사용자에게 다음과 같이 알려주세요: 스킬과 `CLAUDE.md`
변경 사항이 적용되려면 **새 Claude Code 세션을 시작하세요**.
