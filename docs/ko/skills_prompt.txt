# Coast Runtime

이 프로젝트는 격리된 개발 환경을 위해 Coasts(컨테이너화된 호스트)를 사용합니다.
코드 편집 내용은 실행 중인 Coast 내부에 자동으로 반영됩니다 — 파일시스템은
호스트와 컨테이너 간에 공유됩니다.

## Orientation

런타임 명령을 실행하기 전에, 현재 작업 디렉터리와 일치하는 Coast 인스턴스를
확인하세요:

```
coast lookup
```

이 명령은 인스턴스 이름, 포트, URL, 그리고 예시 명령을 출력합니다. 이후의 모든
명령에는 출력에서 확인한 인스턴스 이름을 사용하세요.

Coasts가 어떻게 동작하는지 더 깊은 맥락이 필요하다면, 다음 문서를 읽으세요:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Coast 내부에서 명령을 실행하려면 `coast exec`를 사용하세요. 셸은 워크스페이스
루트(Coastfile이 있는 위치)에서 시작합니다. 먼저 대상 디렉터리로 cd 하세요:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

예시:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

서비스 상태 확인:

```
coast ps <instance>
```

서비스 로그 읽기:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

오류나 익숙하지 않은 동작을 마주치면 Coast 문서에서 검색하세요:

```
coast search-docs "error message or description"
```

이 기능은 시맨틱 검색을 사용합니다 — 자연어로 문제를 설명하면 관련 문서를
찾아줍니다.

## Rules

- 세션에서 첫 런타임 명령을 실행하기 전에 항상 `coast lookup`을 실행하세요.
- 호스트에서 서비스를 직접 실행하지 마세요. 모든 런타임 작업에는 `coast exec`를 사용하세요.
- 호스트에서의 파일 편집은 Coast 내부에 즉시 반영됩니다. 편집 후 파일을 복사하거나 재빌드할 필요가 없습니다.
- `coast lookup`이 어떤 인스턴스도 반환하지 않으면 Coast가 실행 중이 아닐 수 있습니다. `coast run dev-1`을 제안하거나 프로젝트 상태를 위해 `coast ls`를 확인하세요.

=== 스킬 텍스트 끝 ===
