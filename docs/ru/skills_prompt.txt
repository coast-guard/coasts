# Coast Runtime

Этот проект использует Coasts (containerized host) для изолированных окружений разработки.
Ваши правки кода автоматически видны внутри запущенного Coast — файловая система
общая между хостом и контейнером.

## Orientation

Перед выполнением любых runtime-команд выясните, какой экземпляр Coast соответствует вашему
текущему рабочему каталогу:

```
coast lookup
```

Это выводит имя экземпляра, порты, URL и примеры команд. Используйте имя экземпляра
из вывода для всех последующих команд.

Если вам нужен более глубокий контекст о том, как работают Coasts, прочитайте эти документы:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Используйте `coast exec`, чтобы запускать команды внутри Coast. Оболочка стартует в корне
workspace (где находится Coastfile). Сначала перейдите в нужный каталог:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

Примеры:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

Проверьте статус сервисов:

```
coast ps <instance>
```

Прочитайте логи сервисов:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

Если вы сталкиваетесь с ошибками или незнакомым поведением, выполните поиск по документации Coast:

```
coast search-docs "error message or description"
```

Это использует семантический поиск — опишите проблему естественным языком, и он
найдёт релевантную документацию.

## Rules

- Всегда запускайте `coast lookup` перед вашей первой runtime-командой в сессии.
- Не запускайте сервисы напрямую на хосте. Используйте `coast exec` для всех runtime-задач.
- Изменения файлов на хосте мгновенно видны внутри Coast. Вам не нужно
  копировать файлы или пересобирать после редактирования.
- Если `coast lookup` не возвращает ни одного экземпляра, Coast может быть не запущен. Предложите
  `coast run dev-1` или проверьте `coast ls` для состояния проекта.
