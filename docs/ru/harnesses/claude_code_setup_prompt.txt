Вы настраиваете навык Coasts для Claude Code в этом проекте. Выполняйте все
команды из корня проекта.

## Шаг 1: Проверьте наличие Coast CLI

Выполните эту команду:

  coast --version

Если команда `coast` не найдена, остановитесь здесь и сообщите пользователю:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

Не продолжайте, пока CLI не станет доступен.

## Шаг 2: Получите содержимое навыка

Выполните эту команду:

  coast skills-prompt

Вывод состоит из двух частей:

- **Правила Coasts Runtime** — всё от начала до (но не включая) строки, которая начинается с `---`
- **Навык Coasts** — всё, начиная с блока frontmatter `---` и далее (включая строки `---` и всё после них)

Сохраните обе части для следующих шагов.

## Шаг 3: Спросите пользователя

Спросите пользователя: следует ли настроить это **глобально** (доступно в каждом проекте на этой машине) или **только для этого проекта**?

## Шаг 4: Разместите файлы

Если целевой файл уже существует, добавьте раздел Coast Runtime в конец вместо перезаписи — но сначала проверьте, присутствует ли уже раздел `# Coast Runtime`, и если да, пропустите этот шаг.

**Глобальная настройка:**
- Добавьте правила Coast Runtime в `~/.claude/CLAUDE.md`
- Запишите навык Coasts в `~/.claude/skills/coasts/SKILL.md`

**Настройка проекта:**
- Добавьте правила Coast Runtime в `CLAUDE.md` в корне проекта
- Запишите навык Coasts в `.claude/skills/coasts/SKILL.md`

## Шаг 5: Обновите Coastfile

Прочитайте `Coastfile` в корне проекта. Посмотрите на поле `worktree_dir` в
разделе `[coast]`.

Если `.claude/worktrees` **ещё не** указан в `worktree_dir`:

- Если `worktree_dir` — это одна строка, преобразуйте её в массив и добавьте
  `.claude/worktrees`. Например, `worktree_dir = ".worktrees"` становится
  `worktree_dir = [".worktrees", ".claude/worktrees"]`.
- Если `worktree_dir` уже является массивом, добавьте в него `.claude/worktrees`.
- Если `worktree_dir` вообще отсутствует, добавьте
  `worktree_dir = [".worktrees", ".claude/worktrees"]`.

Это относительный путь (внутри корня проекта), поэтому пересоздание контейнера
не требуется — изменения немедленно вступают в силу для отображения worktree.

## Шаг 6: Подтвердите

После размещения файлов покажите пользователю сводку о том, что было создано и где.

Сообщите пользователю: **Начните новую сессию Claude Code**, чтобы изменения
в навыке и CLAUDE.md вступили в силу.
