Вы настраиваете навык Coasts для OpenAI Codex в этом проекте. Выполняйте все
команды из корня проекта.

## Шаг 1: Проверьте наличие Coast CLI

Выполните эту команду:

  coast --version

Если команда `coast` не найдена, остановитесь здесь и сообщите пользователю:

  "Coast CLI не установлен. Сначала установите его: https://coasts.dev/docs/getting-started"

Не продолжайте, пока CLI не станет доступен.

## Шаг 2: Получите содержимое навыка

Выполните эту команду:

  coast skills-prompt

Вывод состоит из двух частей:

- **Правила Coast Runtime** — всё от начала до строки, которая начинается с `---` (но не включая её)
- **Навык Coasts** — всё, начиная с блока frontmatter `---` и далее (включая строки `---` и всё после них)

Сохраните обе части для следующих шагов ниже.

## Шаг 3: Разместите файлы

Настройка Codex всегда выполняется на уровне проекта. Если целевой файл уже существует, добавьте раздел Coast Runtime вместо перезаписи — но сначала проверьте, присутствует ли уже раздел `# Coast Runtime`, и в таком случае пропустите.

- Добавьте правила Coast Runtime в `AGENTS.md` в корне проекта
- Запишите навык Coasts в `.agents/skills/coasts/SKILL.md`
- Создайте `.agents/skills/coasts/agents/openai.yaml` со следующим содержимым:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

## Шаг 4: Обновите Coastfile

Прочитайте `Coastfile` в корне проекта. Посмотрите на поле `worktree_dir` в
разделе `[coast]`.

Если `~/.codex/worktrees` **ещё не** указан в `worktree_dir`:

- Если `worktree_dir` — это одиночная строка, преобразуйте её в массив и добавьте
  `~/.codex/worktrees`. Например, `worktree_dir = ".worktrees"` становится
  `worktree_dir = [".worktrees", "~/.codex/worktrees"]`.
- Если `worktree_dir` уже является массивом, добавьте в него `~/.codex/worktrees`.
- Если `worktree_dir` вообще отсутствует, добавьте
  `worktree_dir = [".worktrees", "~/.codex/worktrees"]`.

Это внешний путь, поэтому, если экземпляр Coast уже запущен для этого
проекта, его необходимо пересоздать с помощью `coast run`, чтобы новое bind-монтирование
вступило в силу. Сообщите об этом пользователю.

## Шаг 5: Подтвердите

После размещения файлов покажите пользователю сводку о том, что было создано и где.

Сообщите пользователю: **Закройте и снова откройте Codex**, чтобы изменения в навыке и AGENTS.md
вступили в силу.
