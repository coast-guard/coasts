Вы настраиваете навык Coasts для Conductor в этом проекте. Запускайте все
команды из корня проекта.

## Шаг 1: Проверка наличия Coast CLI

Выполните эту команду:

  coast --version

Если команда `coast` не найдена, остановитесь здесь и сообщите пользователю:

  "Coast CLI не установлен. Сначала установите его: https://coasts.dev/docs/getting-started"

Не продолжайте, пока CLI не станет доступен.

## Шаг 2: Получение содержимого навыка

Выполните эту команду:

  coast skills-prompt

Вывод состоит из двух частей:

- **Правила Coast Runtime** — всё от начала до строки, начинающейся с `---` (но не включая её)
- **Навык Coasts** — всё, начиная с блока frontmatter `---` и далее (включая строки `---` и всё после них)

Сохраните обе части для следующих шагов.

## Шаг 3: Спросите пользователя

Спросите пользователя: каких провайдеров вы используете в Conductor?

- **Claude** (Anthropic)
- **Codex** (OpenAI)
- **Оба**
- **Другой** — если пользователь называет другого провайдера, выполните
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` и
  `coast docs --path harnesses/README.md`, чтобы определить, где этот провайдер
  ожидает инструкции проекта и навыки, затем следуйте тому же шаблону

## Шаг 4: Разместите файлы

Настройка Conductor всегда выполняется на уровне проекта. Если целевой файл уже существует, добавьте
раздел Coast Runtime вместо перезаписи — но сначала проверьте, есть ли уже
раздел `# Coast Runtime`, и если есть, пропустите.

**Если пользователь выбрал Claude (или оба варианта):**
- Добавьте правила Coast Runtime в `CLAUDE.md` в корне проекта
- Запишите навык Coasts в `.claude/skills/coasts/SKILL.md`

**Если пользователь выбрал Codex (или оба варианта):**
- Добавьте правила Coast Runtime в `AGENTS.md` в корне проекта
- Запишите навык Coasts в `.agents/skills/coasts/SKILL.md`
- Создайте `.agents/skills/coasts/agents/openai.yaml` со следующим содержимым:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

## Шаг 5: Подтверждение

После размещения файлов покажите пользователю сводку того, что было создано и где.

Сообщите пользователю: **Полностью закройте и заново откройте Conductor**, чтобы изменения вступили в силу.
Если команда `/coasts` не появляется сразу, закройте и откройте приложение ещё раз.

## Шаг 6: Обновите Coastfile

Прочитайте файл `Coastfile` в корне проекта. Посмотрите на поле `worktree_dir` в
разделе `[coast]`. Также прочитайте поле `name` — оно понадобится для построения
пути к worktree.

Каталог worktree для Conductor: `~/conductor/workspaces/<name>`, где
`<name>` — это значение поля `name` в Coastfile.

Если этот путь **ещё не** указан в `worktree_dir`:

- Если `worktree_dir` — это одна строка, преобразуйте её в массив и добавьте
  путь Conductor. Например, `worktree_dir = ".worktrees"` становится
  `worktree_dir = [".worktrees", "~/conductor/workspaces/my-app"]`.
- Если `worktree_dir` уже является массивом, добавьте в него путь Conductor.
- Если `worktree_dir` вообще отсутствует, добавьте
  `worktree_dir = [".worktrees", "~/conductor/workspaces/<name>"]`, используя
  фактическое имя проекта.

Это внешний путь, поэтому, если экземпляр Coast уже запущен для этого
проекта, его нужно пересоздать с помощью `coast run`, чтобы новый bind mount
вступил в силу. Сообщите об этом пользователю.

## Шаг 7: Закоммитьте новые файлы

Conductor запускает каждую сессию в изолированном git worktree. Незакоммиченные файлы
не будут переноситься в новые сессии. Закоммитьте только что созданные файлы,
чтобы они были доступны в каждом будущем рабочем пространстве:

  git add CLAUDE.md .claude/ AGENTS.md .agents/ 2>/dev/null; git status

Покажите пользователю, какие файлы добавлены в staging, и попросите подтвердить перед коммитом.
Используйте сообщение коммита вроде `[dh] feat: add Coasts skill for Conductor`.
