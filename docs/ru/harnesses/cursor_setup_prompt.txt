Вы настраиваете навык Coasts для Cursor в этом проекте. Выполняйте все
команды из корня проекта.

## Шаг 1: Проверьте наличие Coast CLI

Выполните эту команду:

  coast --version

Если команда `coast` не найдена, остановитесь здесь и сообщите пользователю:

  "Coast CLI не установлен. Сначала установите его: https://coasts.dev/docs/getting-started"

Не продолжайте, пока CLI не станет доступен.

## Шаг 2: Получите содержимое навыка

Выполните эту команду:

  coast skills-prompt

Вывод состоит из двух частей:

- **Правила Coast Runtime** — всё от начала и до (но не включая) строки, которая начинается с `---`
- **Навык Coasts** — всё от блока frontmatter `---` и далее (включая строки `---` и всё после них)

Сохраните обе части для следующих шагов.

## Шаг 3: Спросите пользователя

Спросите пользователя: следует ли мне настроить это **глобально** (доступно в каждом проекте на этой машине) или **только для этого проекта**?

## Шаг 4: Разместите файлы

Если целевой файл уже существует, добавьте раздел Coast Runtime в конец вместо перезаписи — но сначала проверьте, присутствует ли уже раздел `# Coast Runtime`, и если да, пропустите этот шаг.

**Глобальная настройка:**
- Добавьте правила Coast Runtime в глобальные правила Cursor пользователя
- Запишите навык Coasts в `~/.cursor/skills/coasts/SKILL.md`

**Настройка для проекта:**
- Добавьте правила Coast Runtime в `AGENTS.md` в корне проекта (или в `.cursor/rules/coast.md`, если пользователь предпочитает правила в стиле Cursor — спросите его)
- Запишите навык Coasts в `.cursor/skills/coasts/SKILL.md`

## Шаг 5: Обновите Coastfile

Прочитайте `Coastfile` в корне проекта. Посмотрите на поле `worktree_dir` в
разделе `[coast]`. Также прочитайте поле `name` — оно понадобится вам, чтобы
сформировать путь worktree.

Каталог worktree Cursor — это `~/.cursor/worktrees/<name>`, где `<name>` —
значение поля `name` в Coastfile.

Если этот путь **ещё не** указан в `worktree_dir`:

- Если `worktree_dir` — одиночная строка, преобразуйте её в массив и добавьте
  путь Cursor. Например, `worktree_dir = ".worktrees"` становится
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/my-app"]`.
- Если `worktree_dir` уже является массивом, добавьте в него путь Cursor.
- Если `worktree_dir` вообще отсутствует, добавьте
  `worktree_dir = [".worktrees", "~/.cursor/worktrees/<name>"]`, используя
  фактическое имя проекта.

Это внешний путь, поэтому, если для этого проекта уже запущен экземпляр Coast,
его необходимо пересоздать с помощью `coast run`, чтобы новый bind mount начал
действовать. Сообщите об этом пользователю.

## Шаг 6: Подтвердите

После размещения файлов покажите пользователю сводку о том, что было создано и где.

Сообщите пользователю: **Перезапустите Cursor**, чтобы изменения навыка и правил
вступили в силу.
