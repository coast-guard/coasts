このプロジェクトで T3 Code 用の Coasts スキルを設定します。すべての
コマンドはプロジェクトルートから実行してください。

## Step 1: Coast CLI を確認する

このコマンドを実行します:

  coast --version

`coast` コマンドが見つからない場合は、ここで停止してユーザーに次のように伝えてください:

  "The Coast CLI is not installed. Install it first: https://coasts.dev/docs/getting-started"

CLI が利用可能になるまで続行しないでください。

## Step 2: スキルの内容を取得する

このコマンドを実行します:

  coast skills-prompt

出力は 2 つの部分に分かれています:

- **Coast Runtime rules** — 先頭から `---` で始まる行の直前までのすべて
- **Coasts skill** — `---` の frontmatter ブロック以降のすべて（`---` の行とその後のすべてを含む）

以下のステップのために、両方の部分を保存してください。

## Step 3: ユーザーに確認する

T3 Code は複数のプロバイダーをサポートしています。ユーザーに質問してください: T3 Code では
どのプロバイダーを使っていますか?

- **Codex** (OpenAI)
- **Claude** (Anthropic)
- **Both**
- **Other** — ユーザーが別のプロバイダー名を挙げた場合は、
  `coast docs --path SKILLS_FOR_HOST_AGENTS.md` と
  `coast docs --path harnesses/README.md` を実行して、そのプロバイダーが
  プロジェクト指示とスキルをどこに期待しているかを確認し、その後は同じパターンに従ってください

## Step 4: ファイルを配置する

T3 Code のセットアップは常にプロジェクトレベルです。対象ファイルがすでに存在する場合は、
上書きするのではなく Coast Runtime セクションを追記してください。ただしまず
`# Coast Runtime` セクションがすでに存在するかを確認し、存在する場合はスキップしてください。

**ユーザーが Codex（または両方）を選択した場合:**
- Coast Runtime rules をプロジェクトルートの `AGENTS.md` に追記する
- Coasts skill を `.agents/skills/coasts/SKILL.md` に書き込む（プロジェクトレベル）
- さらに Coasts skill を `~/.codex/skills/coasts/SKILL.md` にも書き込む（グローバルな
  フォールバック — T3 Code はまだプロジェクトレベルの `.agents/skills/` をスキャンしない可能性があります）
- `.agents/skills/coasts/agents/openai.yaml` を以下の内容で作成する:

interface:
  display_name: "Coasts"
  short_description: "Inspect, assign, and open Coasts for this repo"
  default_prompt: "Use this skill when the user wants help finding, assigning, or opening a Coast."

policy:
  allow_implicit_invocation: false

**ユーザーが Claude（または両方）を選択した場合:**
- Coast Runtime rules をプロジェクトルートの `CLAUDE.md` に追記する
- Coasts skill を `.claude/skills/coasts/SKILL.md` に書き込む（プロジェクトレベル）
- さらに Coasts skill を `~/.claude/skills/coasts/SKILL.md` にも書き込む（グローバルな
  フォールバック — T3 Code はまだプロジェクトレベルの `.claude/skills/` を確実にスキャンしない可能性があります）

**Note:** T3 Code はまだ `.agents/skills/` または
`.claude/skills/` のプロジェクトレベルスキルを読み込まない可能性があります。`AGENTS.md` と `CLAUDE.md` のルールは、
それに関係なくすべてのタスクに毎回適用されます。グローバルな `~/.codex/skills/coasts/` フォールバックにより、
Codex プロバイダーがスキルを見つけられるようになります。

## Step 5: Coastfile を更新する

プロジェクトルートの `Coastfile` を読んでください。`[coast]` セクション内の
`worktree_dir` フィールドを確認します。また、`name` フィールドも読んでください — これを
worktree パスの構築に使用します。

T3 Code の worktree ディレクトリは `~/.t3/worktrees/<name>` です。ここで `<name>` は
Coastfile 内の `name` フィールドの値です。

そのパスがまだ `worktree_dir` に含まれて**いない**場合:

- `worktree_dir` が単一の文字列であれば、配列に変換して
  T3 Code のパスを追加します。たとえば、`worktree_dir = ".worktrees"` は
  `worktree_dir = [".worktrees", "~/.t3/worktrees/my-app"]` になります。
- `worktree_dir` がすでに配列であれば、そこに T3 Code のパスを追加します。
- `worktree_dir` がまったく存在しない場合は、
  実際のプロジェクト名を使って
  `worktree_dir = [".worktrees", "~/.t3/worktrees/<name>"]` を追加します。

これは外部パスのため、このプロジェクトですでに Coast インスタンスが実行中の場合は、
新しい bind mount を有効にするために `coast run` で再作成する必要があります。
このことをユーザーに伝えてください。

## Step 6: 確認する

ファイルを配置した後、何が作成され、どこに作成されたかの概要をユーザーに示してください。
グローバルな `~/.codex/skills/coasts/` コピーは、
T3 Code がまだプロジェクトレベルのスキルを読み込まないことへの回避策であると説明してください。

ユーザーに次のように伝えてください: 変更したスキルとルールを有効にするには、
**T3 Code を再起動**してください。
