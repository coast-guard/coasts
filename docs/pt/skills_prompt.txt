# Coast Runtime

Este projeto usa Coasts (host conteinerizado) para ambientes de desenvolvimento isolados.
Suas edições de código ficam automaticamente visíveis dentro do Coast em execução — o sistema de arquivos
é compartilhado entre o host e o contêiner.

## Orientation

Antes de executar quaisquer comandos de runtime, descubra qual instância do Coast corresponde ao seu
diretório de trabalho atual:

```
coast lookup
```

Isso imprime o nome da instância, portas, URLs e comandos de exemplo. Use o nome da instância da saída
para todos os comandos subsequentes.

Se você precisar de mais contexto sobre como os Coasts funcionam, leia estes documentos:

```
coast docs --path concepts_and_terminology/LOOKUP.md
coast docs --path concepts_and_terminology/FILESYSTEM.md
coast docs --path concepts_and_terminology/EXEC_AND_DOCKER.md
coast docs --path concepts_and_terminology/LOGS.md
```

## Running Commands

Use `coast exec` para executar comandos dentro do Coast. O shell inicia na raiz do workspace
(onde o Coastfile está). Faça cd para o diretório de destino primeiro:

```
coast exec <instance> -- sh -c "cd <dir> && <command>"
```

Exemplos:

```
coast exec dev-1 -- sh -c "cd src && npm test"
coast exec dev-1 -- sh -c "cd backend && go test ./..."
coast exec dev-1 -- sh -c "cd apps/web && npx playwright test"
```

## Runtime Feedback

Verifique o status do serviço:

```
coast ps <instance>
```

Leia os logs do serviço:

```
coast logs <instance> --service <service>
coast logs <instance> --service <service> --tail 50
```

## Troubleshooting

Se você encontrar erros ou comportamentos desconhecidos, pesquise na documentação do Coast:

```
coast search-docs "error message or description"
```

Isto usa busca semântica — descreva o problema em linguagem natural e ele irá
encontrar a documentação relevante.

## Rules

- Sempre execute `coast lookup` antes do seu primeiro comando de runtime em uma sessão.
- Não execute serviços diretamente no host. Use `coast exec` para todas as tarefas de runtime.
- Edições de arquivos no host ficam instantaneamente visíveis dentro do Coast. Você não precisa
  copiar arquivos nem reconstruir após editar.
- Se `coast lookup` não retornar instâncias, o Coast pode não estar em execução. Sugira
  `coast run dev-1` ou verifique `coast ls` para o estado do projeto.
